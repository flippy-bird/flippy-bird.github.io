KV cache

参考资料：大模型推理加速：看图学KV Cache

KV cache 只能用于decoder架构，这是因为Decoder有Causal Mask,在推理的时候前面已经生成的字符不需要和后面的字符产生attention, 从而使得前面已经计算的K和V可以缓存起来。

MNN-LLM

python和C++的绑定

  • 直接使用Python提供的C API (#include <Python.h>) 这是Python官方提供的标准方式，用于创建C/C++扩展模块，并没有使用pybind11或者其它的库

在导出MNN模型时(针对LLM模型哈)

具体的C++/python绑定库在pymnn/pip_package/setup.py 这个文件夹，在这个地方产生了 _mnncengine这个库和_tools这个库

跨平台的实现

不同平台的实现代码在 source/backend, 里面可以看到很多后端的实现，如opengl，Vulkan，metal (使用计算着色器实现算子)， qnn

初始化阶段

  1. 创建Interpreter实例：
    1. 使用Interpreter::createFromFile()或Interpreter::createFromBuffer()加载模型
    2. 模型数据会被解析并存储在Content结构中
  2. 创建Session：
    1. 调用Interpreter::createSession()创建会话
    2. 根据配置选择合适的后端（CPU、NNAPI、OpenCL等）
